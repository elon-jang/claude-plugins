🔥 Github 역대 최고 속도 스타 달성한 개발자가 공개한 "AI 코딩 10가지 원칙"

Moltbot(구 Clawdbot) 창시자 Peter Steinberger 의 인터뷰를 보고 적잖이 충격 받았어요.

그는 13년간 70명 규모 회사를 운영하고 매각한 후, 4년 휴식 끝에 2023년 4월 복귀한 베테랑인데요. 그가 말하는 AI 시대 개발은 우리가 아는 것과 완전히 달랐습니다.

그의 시작은 단순했어요.

처음부터 거창한 사업 계획이 없었어요. 단순히 "AI 를 가지고 놀아보자"는 마음으로 시작했고, 외출 중에도 WhatsApp 으로 집 컴퓨터와 대화하고 싶어서 이 도구를 만들었다고 해요.

결정적인 "아하 모멘트"가 있었어요.

여행 중 에이전트에게 음성 메시지를 보냈는데, 음성 지원 기능을 코딩한 적이 없었거든요. 그런데 에이전트가 스스로 Opus 파일 형식을 파악하고, ffmpeg 를 찾아 변환하고, API 키를 찾아 번역까지 해서 답을 보냈대요. 이 순간 그는 에이전트가 "영리하고 수완 좋은 야수"임을 깨달았다고 합니다.

이런 경험을 바탕으로 Peter 가 정리한 AI 코딩 10가지 원칙이에요.

1️⃣ 완벽주의를 버려야 AI 와 일할 수 있어요

• 70명 팀을 이끌며 "내 스타일대로 안 나와도 받아들이는 법"을 배움
• 코드가 내 취향과 100% 일치하지 않아도 동작하면 넘어가는 유연함이 필요
• 이 훈련이 지금 에이전트와 협업할 때 가장 큰 자산이 됨

2️⃣ 에이전트가 스스로 검증할 수 있게 설계해요

• Peter 는 이걸 "루프를 닫는다(Close the loop)"고 표현
• 컴파일, 린트, 실행, 검증까지 에이전트가 직접 수행하도록 시스템 구성
• 사람이 중간에 확인해주는 구조는 병목이 되어 전체 속도를 떨어뜨림

3️⃣ Pull Request 는 죽었고, Prompt Request 시대예요

• 코드 자체보다 그 코드를 만든 프롬프트가 더 중요해짐
• 외부 PR 은 거의 거절하고, 핵심 아이디어만 추출해서 프롬프트로 재활용
• 그의 형제도 같은 방식으로 일한다고 하니, 이미 퍼지고 있는 패턴인 것 같아요

4️⃣ 코드 리뷰 대신 아키텍처 토론만 해요

• Discord 에서도 핵심 팀원과 코드 이야기는 안 함
• 대신 시스템 구조, 큰 결정, 방향성에 대해서만 논의
• 구현 디테일은 에이전트 몫이라는 관점이 팀 전체에 정착됨

5️⃣ 5~10개 에이전트를 동시에 돌려요

• 하나의 작업에 오래 매달리지 않고 여러 작업을 병렬로 큐잉
• 계획 세우고 에이전트에게 맡기고, 바로 다음 에이전트로 이동
• 이렇게 해야 "몰입 상태(Flow state)"를 유지할 수 있다고 함

6️⃣ 계획 단계에 놀라울 정도로 긴 시간을 써요

• 에이전트와 충분히 핑퐁하며 계획을 다듬음
• 도전하고, 수정하고, 반박하고, 만족스러울 때까지 반복
• Codex 선호 이유: Claude Code 는 실행 중 확인 질문을 해서 흐름이 끊김
• 계획이 탄탄하면 실행 단계에서 개입할 필요가 거의 없어짐

7️⃣ 일부러 모호하게 지시해요

• 너무 구체적으로 지시하면 AI 가 그 범위 안에서만 움직임
• 의도적으로 여지를 남겨서 자신이 생각 못한 방향을 발견하게 함
• 제가 직접 해보니 정말 효과 있는 방식이에요. 예상 못한 해결책이 나올 때가 많아요. 물론 항상 이렇게 할 필요는 없습니다

8️⃣ 원격 CI 10분 기다리지 말고 로컬에서 바로 테스트해요

• 원격 CI 파이프라인 대기 시간 10분이 아깝다는 관점
• 에이전트가 직접 로컬에서 테스트를 돌리게 설계
• 피드백 루프를 최대한 짧게 유지해야 반복 속도가 빨라짐

9️⃣ 대부분의 코드는 지루한 데이터 변환일 뿐이에요

• 앱 코드 대부분은 "데이터를 이 형태에서 저 형태로 바꾸는 것"
• 이런 작업에 집착할 필요 없이 에이전트에게 위임
• 에너지는 코드 자체가 아니라 시스템 설계에 집중해야 함

🔟 제품 출시를 좋아하는 사람이 AI 적응에 유리해요

• 알고리즘 퍼즐 푸는 걸 좋아하는 개발자가 오히려 AI 전환에 힘들어함
• 구현 디테일보다 결과물과 출시에 관심 있는 사람이 빨리 적응
• 이건 저도 주변에서 자주 보는 패턴이에요

Peter 의 미래 전망도 흥미로워요

그는 수많은 앱이 사라지고 API 만 남을 거라고 예측해요. MyFitnessPal 을 열어서 입력할 필요 없이, 에이전트에게 음식 사진만 보내면 알아서 칼로리 계산하고 건강 목표를 수정해주는 시대가 온다는 거예요.

여러 논쟁이 있을 수 있지만 결국 Peter 의 10가지 원칙이 말하는 건 하나예요. 완벽주의를 버리고, 코드 리뷰 대신 아키텍처를 논하고, 에이전트가 스스로 검증하게 만들고, 여러 에이전트를 동시에 돌리는 것.

이 모든 게 "개발자가 직접 코드를 짜지 않아도 되는 환경을 만든다"는 방향으로 수렴합니다. 그렇다면 AI 시대 진짜 실력은 코드를 잘 짜는 게 아니라, 코드를 짜지 않고도 문제를 해결하는 판을 설계하는 겁니다.
