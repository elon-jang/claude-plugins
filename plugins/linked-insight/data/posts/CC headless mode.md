Claude Code가 새벽에 혼자 일하고 있으면 어떨까요? Mac 켜놓고 자면 아침에 코드 리뷰가 끝나있는 거죠.

2026년 1월 현재, 실제로 가능합니다. 방법 정리해드릴게요.

---

핵심은 headless mode입니다. Claude Code에 -p 플래그를 붙이면 대화형 인터페이스 없이 바로 실행돼요.

claude -p "코드 리뷰해줘" --output-format json

프롬프트 보내고, 결과 받고, 종료. 터미널에서 한 줄이면 됩니다.

---

문제는 이걸 특정 시간에 자동으로 돌려야 한다는 거죠.

Claude Code 자체에는 아직 스케줄 기능이 없어요. 2025년 7월에 기능 요청이 올라왔는데, 2026년 1월 현재까지 구현되지 않았습니다.

그래서 외부 스케줄러랑 조합해야 해요.

---

첫 번째 방법, runCLAUDErun 앱입니다.

Mac용 네이티브 앱인데요. GUI로 스케줄 설정하면 끝이에요. 코드 한 줄 안 써도 됩니다.

작업 만들고, 시간 선택하고, 저장. 무료고요.

비개발자분들께 강력 추천드립니다.

---

두 번째, cron입니다. 개발자분들은 이게 더 익숙하실 거예요.

crontab -e 열어서 이렇게 추가하면 됩니다.

0 9 * * * /bin/zsh -l -c 'cd ~/projects/my-app && claude -p "코드 리뷰" > ~/logs/review.json'

매일 오전 9시에 자동 실행되는 거죠.

여기서 -l 플래그 꼭 넣으세요. 이거 없으면 환경변수가 안 잡혀서 claude 명령어를 못 찾습니다.

---

세 번째, macOS launchd입니다. cron보다 Mac에서 더 안정적이에요.

plist 파일 만들어서 LaunchAgents에 넣으면 됩니다. 설정이 좀 복잡한 대신 세밀한 제어가 가능해요.

Mac이 잠자기에서 깨어날 때 실행하거나, 특정 조건에서만 실행하는 것도 가능하고요.

---

완전 무인 실행이 필요하시면 두 가지 옵션이 있어요.

하나는 --dangerously-skip-permissions 플래그입니다. 모든 권한 확인을 건너뛰어요.

다른 하나는 --permission-mode acceptEdits 옵션인데요. 편집 권한만 자동 승인하고 나머지는 물어봅니다. 이게 좀 더 안전해요.

CI/CD나 격리된 환경에서는 전자를, 로컬에서는 후자를 추천드립니다.

---

2025년 말에 추가된 기능도 있어요.

Background agent support라고, 에이전트가 백그라운드에서 돌아가는 동안 다른 작업을 할 수 있게 됐습니다.

터미널 하나 잡아먹고 기다릴 필요가 없어진 거죠.

---

실제로 어떻게 쓰냐면요.

저는 매일 밤 11시에 코드 품질 체크 돌리고, 매주 월요일 아침에 주간 리포트 생성합니다.

출근하면 이미 결과가 정리돼있어요. 리뷰만 하면 됩니다.

---

주의할 점 하나요.

Mac이 잠자기 들어가면 스케줄러가 안 돌아갑니다.

시스템 설정에서 "네트워크 접근을 위해 깨우기" 켜두시고, pmset으로 예약 깨우기 설정해두세요.

그래야 새벽에도 확실히 실행됩니다.

---

정리하면 이렇습니다.

쉬운 거 원하시면 runCLAUDErun
세밀한 제어 원하시면 launchd
빠르게 쓰고 싶으시면 cron

어떤 방법이든 핵심은 claude -p 플래그예요. 이것만 알면 어디든 붙일 수 있습니다.

여러분은 Claude Code 자동화 어떻게 쓰고 계세요?
